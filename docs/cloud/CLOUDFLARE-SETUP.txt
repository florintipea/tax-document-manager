TaxDoc — Optional Cloudflare Setup
==================================

Put Cloudflare in front of your Render service for edge DDoS protection,
WAF rules, and Bot Fight Mode. TaxDoc works without Cloudflare; this is
optional hardening for production.

Prerequisites
-------------
- A custom domain (e.g. app.yourdomain.com)
- Render service already deployed (e.g. taxdoc-beta.onrender.com)


Step 1 — Add site to Cloudflare
--------------------------------
  1. Sign up / log in at https://dash.cloudflare.com
  2. Add site → enter your domain
  3. Choose Free plan (sufficient for basic WAF + DDoS)
  4. Update nameservers at your registrar to Cloudflare's NS records


Step 2 — DNS (CNAME to Render)
-------------------------------
  Type: CNAME
  Name: app (or @ for apex — use CNAME flattening if supported)
  Target: taxdoc-beta.onrender.com
  Proxy status: Proxied (orange cloud)

In Render dashboard:
  - Settings → Custom Domains → add app.yourdomain.com
  - Verify DNS; Render issues TLS for your domain


Step 3 — SSL/TLS mode
---------------------
  Cloudflare → SSL/TLS → Overview → Full (strict)

This encrypts traffic Cloudflare ↔ Render with a valid certificate on Render.


Step 4 — Trust Cloudflare client IP in TaxDoc (optional)
---------------------------------------------------------
When proxied, set in Render environment:

  TRUST_CLOUDFLARE=true

Then redeploy. TaxDoc uses CF-Connecting-IP for rate limiting and audit logs.

Important:
  - Direct access via *.onrender.com still works (falls back to X-Forwarded-For)
  - Only enable TRUST_CLOUDFLARE when all production traffic goes through Cloudflare
  - Do not enable if users can bypass Cloudflare and spoof CF-Connecting-IP


Step 5 — Recommended Cloudflare settings
-----------------------------------------
Security → Settings:
  - Security Level: Medium (or High for public beta)
  - Bot Fight Mode: On (Free plan)

Security → WAF → Managed rules:
  - Enable Cloudflare Managed Ruleset (Free includes basic rules)

Rate limiting (optional, paid or limited free):
  - Rule: URI Path contains /api/auth/login → 10 req/min per IP
  - Rule: URI Path contains /api/auth/register → 5 req/min per IP

Caching:
  - Do NOT cache /api/* or authenticated pages
  - Static assets (_next/static) can use standard caching


Step 6 — Verify
---------------
  1. Visit https://app.yourdomain.com/api/health → ok: true
  2. As admin: /api/health?admin=1 → trustCloudflare: true (if env set)
  3. Test login and 2FA still work behind Cloudflare


Troubleshooting
---------------
- 525 SSL handshake failed → set SSL mode to Full (strict), verify Render custom domain cert
- Redirect loops → check NEXTAUTH_URL and APP_URL match your Cloudflare domain
- Rate limits too aggressive → adjust Cloudflare rules or TaxDoc REDIS_URL for app-level limits

See also: CLOUD-HOSTING.txt (Render Starter, REDIS_URL)
